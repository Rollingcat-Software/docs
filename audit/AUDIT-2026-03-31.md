## CRITICAL — Fix Immediately

### C1. Spring Security Authorization Bypass (CVE-2025-41232) [FIXED]
- **Where:** `identity-core-api` → `spring-security-core:6.4.5`
- **Impact:** Authorization bypass — attackers may access protected endpoints
- **Fix:** Upgrade to `spring-security-core:6.4.6+`
- **Priority:** IMMEDIATE

### C2. Spring Security Policy Bypass (CVE-2026-22732)
- **Where:** `identity-core-api` → `spring-security-web:6.4.x`
- **Impact:** Security policy bypass and potential memory leak
- **Fix:** Upgrade to `spring-security-web:6.5.9+` (requires Spring Security 6.5 line — awaiting upstream compatibility)
- **Priority:** IMMEDIATE
- **Status:** Awaiting upstream release compatible with Spring Boot 3.4.x

### C3. JWT Tokens Exposed in Config (Semgrep) [FIXED]
- **Where:** `fivucsas/.claude/settings.local.json` lines 152, 160, 165-169
- **Impact:** 7 JWT tokens in a settings file — may be test tokens but risky if committed
- **Fix:** Remove tokens, use env vars. Verify file is in `.gitignore`
- **Priority:** HIGH

### C4. Shell=True in subprocess (Semgrep) [FIXED]
- **Where:** `scripts/add_diagrams_to_docx.py:50`
- **Impact:** Command injection if input is not sanitized
- **Fix:** Use `subprocess.run(['cmd', 'arg'], shell=False)`

## Fixes Applied (2026-03-31)

All findings except C2 (CVE-2026-22732, awaiting upstream Spring Security 6.5.x compatibility) have been remediated.

### Dependency Upgrades (identity-core-api)
- Spring Boot 3.4.5 → 3.4.7
- Spring Security 6.4.5 → 6.4.13 (fixes C1, H3)
- Tomcat Embedded 10.1.40 → 10.1.53 (fixes H1 — 6 CVEs)
- Netty 4.1.119.Final → 4.1.132.Final (fixes H2 — 3 CVEs)
- PostgreSQL JDBC 42.7.5 → 42.7.7 (fixes H5)
- Commons-IO 2.7 → 2.21.0 (fixes H6)

### Secrets Removal
- 7 JWT tokens removed from `.claude/settings.local.json` (fixes C3)

### Code Fixes
- `shell=True` → `shell=False` in `scripts/add_diagrams_to_docx.py` (fixes C4)
- Console.log string concatenation → template literals in `auth-test/app.js` (fixes M5)

### Infrastructure Hardening
- Docker Compose: `security_opt: [no-new-privileges:true]` added to all services, `read_only: true` where safe (fixes H8)
- Nginx: security headers moved into location blocks to prevent override (fixes M2)
- Dockerfiles: consecutive RUN instructions consolidated, `--no-install-recommends` added to apt-get (fixes M4)
- SRI integrity hashes added to all CDN `<script>` tags in `auth-test/index.html` (fixes M1)

### Remaining
- **C2 (CVE-2026-22732)**: Spring Security Policy Bypass — requires Spring Security 6.5.9+ which is not yet compatible with Spring Boot 3.4.x. Will be addressed when upstream releases a compatible version.
